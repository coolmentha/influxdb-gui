# 结果集默认注入 LIMIT + 硬上限

InfluxDB 1.x 的 `/query` 一次性返回全部匹配点，时序数据轻易到百万级。为了 UI 不被拖死，v1 采用"未显式带 LIMIT 的查询由客户端自动追加 `LIMIT N`（默认 1000，可调）+ 无论如何硬上限 100k 行"的策略。

这是 DataGrip / DBeaver 的通行做法，用户可在 UI 顶栏改 LIMIT 或勾"无限制"。硬上限触发时中断并提示"结果过大"。

## Considered Options

- **不改用户查询、仅前端虚拟滚动**：内存仍会爆，只延迟崩溃。
- **后端流式分块**：最抗压，但 1.x 没有原生流式游标，实现成本高，留到 v3+ 再谈。

## Consequences

- 用户可能会误以为看到的是全量数据。UI 必须在结果集顶部持续显示"已限制到 N 行"的红/黄提示与还原开关。
